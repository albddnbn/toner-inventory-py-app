About the Printer Inventory App:

The application GUI was built using the PyQt Python package, which offers many advantages over other Python GUI libraries like tkinter. The first version of this app actually was made using tkinter, but just after seeing how the GUI alone could be improved through the use of PyQt, I was convinced to make the transition.

IT personnel can use this app along with a USB/Bluetooth barcode scanner to scan inventory and automatically update the table in the database that the app is connected to.

Users can also easily manipulate items in the database using the application's GUI. Most common operations are possible, including the addition, deletion, and editing of all properties of the different printer-related items.

Item types:

Every item in the database (row in the 'inventory' table) is assigned an item 'type'. For example - Dell c3760 Magenta Toner would be assigned a 'type' of 'toner'.

There are currently 4 item types:

  • toner - probably the most common item type that will be held in inventory.
  • ink
  • drum - imaging units, imaging drums
  • maintkit - maintenance kits
  • waste - toner collection units, waste containers
  • roller - pickup rollers, transfer rollers
  • None or null - items type hasn't been assigned yet

The application will try to parse each item's type from it's descriptions (utils/fillout_columns.py). The descriptions can be found inside the json responses from the UPC lookup API, and there are usually at least a couple descriptions to look through for item type, and other column values.

Features:


Scan inventory into database

To begin scanning inventory, click the barcode icon on the secondary toolbar:

Once the scan input window opens, make sure the text cursor is inside of the window's textbox and begin to scan items.

Be sure to pay attention to either the LED indicators, or the auditory aids while scanning inventory:

  • visual - while the green LED is lit, the user is ok to continue scanning.
    • if the red LED is lit, that means that the app has failed to look up the item that was just scanned and the scanning process has been frozen until the user acknowledges the failure and enters a few notes about the missed item to help with manual addition to the database after the scan has been completed.
  • auditory - after every successful lookup, the user will hear a 'pleasant' sounding chime sound.
    • after a FAILED lookup, the user will hear an unpleasant but hopefully not annoying buzzer alert sound.


Once the user has finished scanning inventory, they can click the Save button to save the scan list to a text file. They will be prompted to do so even if they click 'Exit' without clicking save, and even if the user does choose 'No' they would not like to save the scan list, the scan list will still be saved using a predetermined filename format.

After a FAILED LOOKUP:

As stated above, the user will hear an alert sound and see that the red LED is lit. No more items can be scanned in while the red LED is lit.

To resume scanning inventory, the user must:

Click the 'Enter Item Notes' button, and enter a sentence or two of information that will help to remember what item the scanned code is after the scan has been completed. Since addition of this item to database did NOT OCCUR, the user has to add the item in through the GUI manually. Sometimes it can be difficult to remember what item is linked to a failed code, which is where the reminder notes come in.



When the scan is complete, the user can check the 'output' directory for a complete list and count of any missed items, along with the user notes.

How item lookups work:

When a barcode scanner is used to scan an item, the app will check the database for the item first, and then send out to the api if the item is not found locally.

If the item is found locally, an UPDATE SQL statement is sent to the server which adds 1 to the item's stock value.

If the item is not found locally, the app will send a GET request to the configured api (upcitemdb.com).

Configuration:

Eventually, the easiest way to configure application settings will be through Settings > View all settings (the settings dialog).

The settings dialog is still in development, but at this point it does allow the user to change the more important settings.

App variables:

apiurl is set to the UPC Lookup API that's being used

campus set to wherever this instance of the app is being used

icon the application icon, set to Delaware Tech icon. Shows up as the icon for basically all of the windows and system notifications.

searchmode specifies the default/starting search mode for the application, as well as the current search mode while the app is running.

theme specifies the default/starting theme for the app, along with the current theme while the app is running.


Database variables: specify database connection parameters


Directories

json specifies where the .json files are stored (results of calls to item lookup API)

Logs

main the main application log; records all interactions with database, and other important events.

Preferences

mute 1 means muted, 0 means not muted.

safetynet specifies whether the application will close directly when the user clicks 'X', or whether there will be a popup asking for confirmation to close the app.

0 means there is no safety net, so application will close immediately after clicking 'X'.

selectedcolumns and selectedindices specify which columns from the database will be shown when a spreadsheet is viewed using the application.

Sounds

success the sound that plays for positive events during an inventory scan. Only tested with .wav

fail negative events during inventory scan. Only tested with .wav

The app's configuration file is located in the config directory: config.yaml.

Application themes:

There are going to be a lot of choices for color schemes and themes for the app, mostly due to the fact that PyQt application styling comes from QSS stylesheets. QSS is basically CSS, for Qt. The styles seen throughout this guide are: ElegantDark, DTCC, and Ubuntu.

Most or all of the QSS stylesheets used in this app have been adapted from https://github.com/GTRONICK/QSS


You can change application theme by using the main toolbar: Settings > Change theme


If you want to make a theme for the app, the easiest way may be to use this QSS Stylesheet Editor from Github: https://github.com/hustlei/QssStylesheetEditor

There is some base styling and formatting, specific to this application but not specific to a certain theme that needs to be extracted from the most developed theme stylesheet (ElegantDark.QSS). Then, this base stylesheet can be applied to any QSS stylesheet to make it a theme choice in this application.

Areas the application could be improved:

Although I'm sure there are many areas the app can be improved that I'm not aware of, several key areas that I am aware of are listed below.

  1. Parsing of item details from the API's json responses. It would be great to find a way to parse the item details like color, item type, compatible printers, etc. more effectivelty.

    At this point, there isn't any method to parse compatible printers. It might be possible (a fraction of the time) using the item names and descriptions.

  2. Better looking widgets on dashboard for different item types. I like the toner and ink radials, but imaging units, fusers, waste containers, could all use something other than the standard battery level indicator that's being used on the dashboard now.
